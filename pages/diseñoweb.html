Etiquetas y sintaxis de código

Hay etiquetas cerradas y abiertas

  • Las etiquetas cerradas encierran un contenido, por lo general texto.
  • Las etiquetas abiertas sirven, entre otras cosas, para incluir elementos como imágenes, líneas, entre otros. Las etiquetas abiertas también se conocen como “autocontenidas”, o que “se cierran en sí mismas”.

Etiquetas autocerradas

  • < area />: define un área en la que se puede hacer clic dentro de un mapa de imagen.
  • < base />: especifica una URL base para las URL relativas de un documento.
  • < br />: inserta un salto de línea dentro del texto o contenido.
  • < circle />: Define una ruta circular dentro de SVG (Gráficos vectoriales escalables).
  • < col />: Especifica propiedades de columna para tablas HTML.
  • < embed />: incorpora contenido externo, como multimedia o complementos.
  • < hr />: Crea un salto temático o una línea horizontal.
  • < img />: incrusta una imagen en el documento.
  • < input />: Define un elemento de entrada del usuario.
  • < link />: vincula recursos externos como hojas de estilo o íconos.
  • < meta />: proporciona metadatos sobre el documento.
  • < param />: Especifica parámetros para complementos dentro de elementos de objeto.
  • < path />: Define una ruta vectorial dentro de SVG (Gráficos vectoriales escalables).
  • < source />: Especifica múltiples recursos multimedia para elementos multimedia.
  • < track />: Proporciona pistas de texto para elementos multimedia como '< video>' o '< audio>'.
  • < wbr />: Define una oportunidad de salto de palabra dentro del texto.
  • < command />: Define un botón de comando dentro de un menú.
  • < keygen />: Genera un par de claves para formularios utilizados en criptografía.
  • < menuitem />: Define un elemento dentro de un '< menu>'.
  • < frame />: Define una subventana (obsoleta en HTML5).

HTML: Estructura y elementos básicos

HTML5 incorpora etiquetas semánticas que no sólo funcionan como contenedores, sino que también definen qué tipo de contenido van a tener.

Contenedores no semanticos

Para organizar el contenido de nuestros documentos HTML, debemos tener en cuenta que los elementos (textos e imágenes) tienen que estar contenidos en un contenedor. El div es un contenedor en bloque mientras que el span es un contenedor en línea muy útil para el trabajo con textos.

Enlaces de interes

CSS

CSS en el HTML

CSS (Cascading Style Sheets) es un lenguaje web para aplicar formato visual (color, tamaño, separación y ubicación) al HTML. Es así que puedes cambiar por completo el aspecto de cualquier etiqueta HTML.

Sintaxis

✓ Cada declaración CSS está formada por un par propiedad: valor; ✓ No se ve afectado por el espacio en blanco. Las propiedades se pueden escribir de corrido o una debajo de la otra. ✓ Siempre que la propiedad represente un número, el valor debe indicar en qué unidad se expresa.

reseteo CSS

👉 Los reset CSS contienen, en su código fuente, definiciones para propiedades problemáticas que los diseñadores necesitan unificar desde un principio. Por ejemplo, la mayoría de navegadores establece un margen por defecto entre el contenido de la página web y su propia ventana, cuyo valor varía de un navegador a otro. 👉Para subsanar esa diferencia, los diseñadores y las diseñadoras de sitios webs suelen declarar la siguiente línea al comienzo de sus hojas de estilo:

Box Modeling y Flexbox

box Modeling

El modelo de caja (Box Model) en CSS es un concepto fundamental que describe cómo los elementos HTML son representados en el navegador. Cada elemento se ve como una caja rectangular que se compone de varios componentes, los cuales determinan su tamaño y cómo interactúan con otros elementos.

  1. Contenido (Content):Contenido (Content):
    • Esta es el área donde se coloca el contenido, como texto, imágenes o otros elementos incrustados.
    • El tamaño del contenido se puede ajustar utilizando las propiedades width y height.
  2. Relleno (Padding):
    • El relleno es el espacio entre el contenido y el borde de la caja.
    • Puedes establecer el relleno con las propiedades padding-top, padding-right, padding-bottom, padding-left o la propiedad abreviada padding.
    • El relleno añade espacio dentro de la caja, aumentando su tamaño total.
  3. Borde (Border):
    • El borde rodea el contenido y el relleno. Es una línea que puede tener grosor, color y estilo.
    • Puedes establecer el borde con las propiedades border-width, border-style, border-color o la propiedad abreviada border.
    • El borde también añade espacio alrededor del contenido y el relleno, aumentando el tamaño total de la caja.
  4. Margen (Margin):
    • El margen es el espacio exterior alrededor de la caja. Se usa para separar la caja de otros elementos.
    • Puedes establecer el margen con las propiedades margin-top, margin-right, margin-bottom, margin-left o la propiedad abreviada margin.
    • A diferencia del relleno y el borde, el margen no añade espacio dentro de la caja, pero sí fuera de ella.

Diagrama de Box Model y valores

Por defecto, el tamaño de una caja en CSS se calcula sumando el contenido, el relleno y el borde. Sin embargo, puedes cambiar este comportamiento usando la propiedad box-sizing:

  • box-sizing: content-box; (valor por defecto): El tamaño de width y height solo incluye el contenido. El relleno y el borde se añaden fuera de estos valores.
  • box-sizing: border-box;: El tamaño de width y height incluye el contenido, el relleno y el borde. Esto hace que el tamaño total de la caja sea más fácil de controlar.

Flexbox

Flexbox, o "Flexible Box Layout", es un modelo de diseño en CSS que permite crear estructuras de diseño complejas y adaptativas con facilidad. Es especialmente útil para distribuir el espacio dentro de un contenedor y alinear elementos en filas y columnas.

Componentes Principales de Flexbox

  1. Contenedor Flex (Flex Container):
    • El contenedor flex es un elemento padre al que se le aplica la propiedad 'display: flex' o 'display: inline-flex.'
    • Los elementos hijos directos de este contenedor se convierten en elementos flex (flex items).
  2. Elementos Flex (Flex Items):
    • Los elementos flex son los elementos hijos de un contenedor flex.
    • Se distribuyen y alinean según las propiedades aplicadas al contenedor y a ellos mismos.

Propiedades del Contenedor Flex

  1. display:
    • display: flex;: Define un contenedor flex de bloque.
    • display: inline-flex;: : Define un contenedor flex en línea.
  2. flex-direction:
    • row: Los elementos se colocan en una fila, de izquierda a derecha.
    • row-reverse: Los elementos se colocan en una fila, de derecha a izquierda.
    • column: Los elementos se colocan en una columna, de arriba a abajo.
    • column-reverse: Los elementos se colocan en una columna, de abajo a arriba.
  3. flex-wrap:
    • nowrap: Los elementos no se envuelven (valor por defecto).
    • wrap: Los elementos se envuelven en múltiples líneas, de arriba a abajo.
    • wrap-reverse: Los elementos se envuelven en múltiples líneas, de abajo a arriba.
  4. justify-content:
    • Define la alineación de los elementos flex a lo largo del eje principal.
    • Valores: flex-start, flex-end, center, space-between, space-around, space-evenly.
  5. align-items:
    • Define la alineación de los elementos flex a lo largo del eje transversal (perpendicular al eje principal).
    • Valores: flex-start, flex-end, center, baseline, stretch.
  6. align-content
    • Define la alineación de las líneas flex cuando hay espacio adicional en el contenedor.
    • Valores: flex-start, flex-end, center, space-between, space-around, stretch.

Grid

CSS Grid Layout es una especificación de CSS que permite diseñar páginas web y aplicaciones de una manera más flexible y eficiente. CSS Grid Layout proporciona un sistema de cuadrícula bidimensional para colocar elementos en filas y columnas, lo que facilita la creación de diseños complejos sin necesidad de utilizar muchos div y flotadores.

Propiedades del padre

Comenzamos aplicando la propiedad “display: grid” al elemento padre. Luego podemos usar lo siguiente para crear la estructura principal.

  • grid-template-columns :Establece el TAMAÑO de cada columna (col 1, col 2...).
  • grid-template-rows :Establece el TAMAÑO de cada fila (fila 1, fila 2...).
  • grid-template-areas :Indica la disposición de las áreas en el grid. Cada texto entre comillas simboliza una fila.
  • grid-column-gap :Establece el TAMAÑO de los huecos entre columnas (líneas verticales).
  • grid-row-gap :Establece el TAMAÑO de los huecos entre filas (líneas horizontales).

GRID POR ÁREAS

Es posible indicar el nombre y la posición concreta de cada área de la cuadrícula. Utilizar la propiedad grid-template-areas, donde debemos especificar el orden de las áreas. Luego, en cada item hijo, utilizamos la propiedad grid-area para indicar el nombre del área del que se trata De esta forma, es muy sencillo crear una cuadrícula altamente personalizada en apenas unas cuantas líneas de CSS

De ésta forma, se aproxima a lo que queremos inicialmente, solo nos falta darle unas decoraciones

Meta viewport

Las páginas optimizadas para diferentes dispositivos deben incluir la etiqueta < meta> viewport en el encabezado del documento HTML. Una etiqueta < meta> viewport da al navegador las instrucciones sobre cómo controlar las dimensiones y el ajuste a escala de la página.

  • Usa la etiqueta < meta> viewport para controlar el ancho y el ajuste de la ventana de visualización del navegador.
  • Incluye width=device-width para hacer coincidir el ancho de la pantalla en píxeles independientes del dispositivo.
  • Incluye initial-scale=1 para establecer una relación de 1:1 entre los píxeles CSS y los píxeles independientes del dispositivo.

< meta name="viewport" content="width=device-width, initial-scale=1.0"/>

BEM y Bootstrap

Metodología BEM

La metodología BEM es una forma poderosa y efectiva de organizar y nombrar clases CSS para facilitar la creación de componentes reutilizables, legibles y mantenibles. Al seguir las convenciones de BEM, se puede gestionar mejor el CSS en proyectos grandes y complejos, evitando conflictos de nombres y mejorando la claridad del código.

Conceptos básicos de BEM

  1. Block (Bloque):
    • Representa un componente autónomo de la interfaz de usuario que tiene sentido por sí mismo. Puede ser un botón, una tarjeta, un menú, etc.
    • Se define con una clase única.
    • Ejemplo: menu, button, card.
  2. Element (Elemento):
    • Representa una parte de un bloque que no tiene sentido por sí misma y está ligada al bloque.
    • Se define con el nombre del bloque seguido de dos guiones bajos y el nombre del elemento.
    • Ejemplo: menu__item, button__icon, card__title.
  3. Modifier (Modificador):
    • Representa una variación o estado del bloque o elemento.
    • Se define con el nombre del bloque o elemento seguido de dos guiones y el nombre del modificador.
    • Ejemplo: menu--horizontal, button--primary, card__title--large.

Variables, importaciones y partials

Variables

Las variables en SCSS te permiten almacenar valores que puedes reutilizar en tu hoja de estilos. Esto facilita la gestión de colores, tamaños de fuente, márgenes, y otros valores que pueden cambiar.

Importaciones

SCSS permite dividir el código CSS en varios archivos y luego importarlos en un solo archivo principal. Esto se hace utilizando la directiva @import. Las importaciones ayudan a mantener el código organizado y modular.

Ejemplo de Importaciones: Supongamos que tenemos dos archivos SCSS: _variables.scss y _mixins.scss.

Partials

Los partials son archivos SCSS que se importan en otros archivos SCSS. Se nombran con un guion bajo (_) al principio del nombre del archivo para indicar que son partials. Los partials no se compilan en archivos CSS independientes, sino que se combinan en el archivo principal que los importa.

Ejemplo de Partials: Usamos los mismos archivos _variables.scss y _mixins.scss del ejemplo anterior. Estos archivos son partials y se importan en style.scss.

  1. _variables.scss y _mixins.scss son partials porque tienen un guion bajo al principio de sus nombres.
  2. En main.scss, importamos estos partials sin incluir el guion bajo ni la extensión del archivo.

Mixins

  1. Concepto:

    En SCSS, los mixins son bloques de código reutilizable que pueden incluirse en otros bloques de código CSS utilizando @include. Los mixins pueden aceptar argumentos, lo que permite una mayor flexibilidad y reutilización.

En resumen, en SCSS, @extend se utiliza para compartir propiedades CSS entre selectores para evitar la duplicación de código, mientras que los mixins (@mixin y @include) se utilizan para reutilizar bloques de código con la posibilidad de pasar argumentos para una mayor flexibilidad.

MAPS

Con Sass, se pueden definir mapas - el término en Sass para los arreglos asociativos, hashes o incluso objetos JavaScript. Un mapa es una estructura de datos que asocia claves a valores. Tanto las claves como los valores, pueden ser de cualquier tipo de dato, incluyendo mapas, aunque yo no recomendaría usar tipos de datos tan complejos como claves de un mapa, solo por salud mental. Los mapas deberían estar escritos de la siguiente manera:

  1. poner un espacio después de los dos puntos (:);
  2. la llave de apertura (() debe ir en la misma línea que los dos puntos (:);
  3. poner las claves entre comillas si son cadenas (lo que representa el 99% de los casos);
  4. cada par clave/valor debe ir en su propia línea;
  5. poner coma (,) al final de cada par clave/valor;
  6. poner coma final (,) en el último elemento para que sea más sencillo añadir, eliminar o reordenarlos;
  7. la llave de cierre ()) debe ir en una línea nueva;
  8. no poner espacio o línea nueva entre la llave de cierre ()) y el punto y coma (;).

Animaciones

Animaciones son una forma de mostrar la interacción entre los elementos de un diseño y el usuario. Las animaciones pueden ser de diferentes tipos, como transiciones, efectos de movimiento, y efectos de color.

SEO on-page

El SEO On-Page involucra las optimizaciones que podemos controlar dentro de nuestro sitio web. Dentro del SEO on-page podemos incluir

  1. Meta description

    La Metaetiqueta de descripción de una página, ofrece a los buscadores un resumen del contenido de la misma. puede contener una frase o dos, incluso un párrafo breve.

  2. Keywords (palabras clave)

    Las palabras clave son palabras o frases que se usan para buscar un sitio web en los motores de búsqueda. Hay que identificar los términos que buscan los usuarios, y con los cualkes se desea que un sitio web sea encontrado por los motores de busqueda. Hay páginas que generan estas palabras.

  3. Títulos de páginas

    Lo ideal sería crear un título único para cada página del sitio. El título de tu página principal puede inclusir el nombre de tu sitio web o negocio, y podría inclusir otra información importante.
